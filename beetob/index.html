Recherche & Enrichissement

Claude cherche des prospects réels sur le web (Pages Jaunes, Pappers, sites officiels) à partir d'une ville ou d'un code postal, puis les enrichit avec nom du dirigeant, SIREN, fournisseur actuel.

Comment ça marche : Cette fonction utilise web_search de l'API Claude (~10$/1000 recherches). Une recherche complète sur une ville utilise 5-10 requêtes web (~0,05-0,10 €) et te ramène 10-20 prospects qualifiés en 60-90 secondes. Tu valides chaque prospect avant import.

Documentation

Mode d'emploi

Démarrage rapide

  1. Configure ton API Anthropic : clic sur "API non configurée" en haut à droite. Récupère ta clé sur console.anthropic.com (gratuit pour créer un compte, mais nécessite un moyen de paiement).
  2. Importe ton CSV de prospects (onglet "Données"). Coche "Scorer avec IA" pour que Claude analyse chaque ligne.
  3. Tableau de bord : vois ce qu'il faut traiter aujourd'hui. Sur chaque action, le bouton "Personnaliser ✨" demande à Claude d'adapter le modèle au prospect.

Les 5 fonctions IA

  • Scoring automatique : à l'import, Claude analyse chaque ligne et propose un score /100 + un commentaire
  • Brief express (fiche prospect) : 5 lignes de synthèse stratégique sur le prospect
  • Personnaliser cet email : Claude réécrit le modèle générique en intégrant les notes prospect
  • Analyser une réponse : tu colles la réponse du prospect, Claude détecte le sentiment et propose la prochaine action
  • Revue de pipeline : analyse synthétique de l'état du pipeline et top 3 actions de la semaine

Coût et sécurité

Ta clé API est stockée uniquement sur ton appareil (jamais transmise à Anthropic ni à un serveur tiers). Coût moyen : 0,01 à 0,03 € par appel IA avec Claude Sonnet 4.5. Tu peux suivre ta conso dans l'onglet Données.

Pour révoquer ta clé en cas de doute : connecte-toi sur console.anthropic.com et désactive la clé. Elle ne marchera plus dans l'outil.

Limites

  • Pas d'envoi automatique : l'IA prépare, tu valides et envoies via Outlook ou LinkedIn
  • Pas de récupération des réponses : tu copies-colles les réponses dans les notes, l'IA les analyse
  • Pas de tracking ouverture/clic : pour ça il faut un outil tiers (Splio, Mailchimp)

🏠 Connexion LM Studio

LM Studio doit tourner sur le PC depuis lequel tu utilises cette appli. Serveur démarré + "Enable CORS" coché.

Par défaut : http://localhost:1234 (LM Studio sur ce PC)
Préparer LM Studio :
  1. Télécharge LM Studio sur lmstudio.ai
  2. Télécharge un modèle (recommandé : mistral-nemo-instruct ou llama-3.1-8b-instruct)
  3. Onglet Developer → charge le modèle → Start Server
  4. Settings → Enable CORS ✓
  5. Vérifie l'URL ci-dessus puis clique "Tester et enregistrer"

🏢 Connexion Microsoft 365 Copilot

API Copilot Chat (preview) via Microsoft Graph. Nécessite une licence M365 Copilot et un hébergement Entra (Power Pages / site https).

Connexion automatique (MSAL) — recommandée en production
À savoir :
  • La connexion MSAL exige une app Entra de type SPA avec l'URL de redirection = l'adresse exacte de cette page (ton portail Power Pages)
  • Autorisations Graph à accorder (consentement admin probable) : Sites.Read.All, Mail.Read, People.Read.All, OnlineMeetingTranscript.Read.All, Chat.Read, ChannelMessage.Read.All, ExternalItem.Read.All
  • Rien de tout ça ne fonctionne depuis un fichier local (file://) — ni MSAL, ni l'appel Graph (CORS). Destiné à l'hébergement Power Pages
  • L'API Copilot ne renvoie que du texte (souvent markdown) : le scoring JSON strict peut échouer
